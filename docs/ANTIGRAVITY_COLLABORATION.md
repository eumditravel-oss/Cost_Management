# Google Antigravity 협업 프로토콜

## 역할 분담

- Google Antigravity (약 70%): 지정된 미세 작업 단위 구현, 단위 테스트, 작업 증거 작성.
- Codex (약 30%): 작업 분해, 데이터 정합성·권한·계산 규칙 검토, 변경 검수, 통합 테스트 및 Phase 승인 보고.

Codex는 Antigravity의 구현 결과를 확인하기 전에는 해당 작업을 완료로 인정하지 않는다.

## 작업 단위

Antigravity는 한 번에 하나의 `AG-12.x` 작업만 수행한다. 다음 작업으로 넘어가기 전에 반드시 현재 작업의 커밋, 테스트 출력, 변경 파일 목록, 미확인 사항을 `docs/antigravity-handoffs/AG-12.x.md`에 기록한다.

| ID      | Antigravity 구현 범위                    | Codex 검수 기준                        |
| ------- | ---------------------------------------- | -------------------------------------- |
| AG-12.1 | 원장 화면의 기준정보 선택·서버 저장 연결 | 권한, 회사/현장 소속, 계산 재검증      |
| AG-12.2 | 원장 조회·중복 후보 표시·오류 셀         | 중복 오탐/누락, 오류 원문 비노출       |
| AG-12.3 | 템플릿·자동완성·최근값·임시저장 복구     | 개인정보 제외, 사용자별 분리           |
| AG-12.4 | 저장 필터·개인 컬럼 설정                 | 권한 우회 없음, 설정 격리              |
| AG-12.5 | Excel 내보내기                           | 실제 데이터만, 원본 xls 실행/수정 금지 |

## Antigravity 금지사항

- 원본 `거래명세표 양식.xls` 실행·수정·매크로 실행 금지.
- 주민번호, 계좌번호, 주소, 전화번호를 테스트·로그·스크린샷에 넣지 않음.
- 계산·상태 전이·권한을 추정해 확정하지 않음.
- `main` 또는 `master` 직접 push, force push, PR 생성, 배포 금지.
- 테스트하지 않은 결과를 통과로 기록하지 않음.

## 자동 검증과 검수 트리거

1. Antigravity는 `antigravity/AG-12.x-*` 브랜치에 push한다.
2. GitHub Actions가 lint·typecheck·test·build를 자동 실행한다.
3. Antigravity는 핸드오프 문서에 `READY_FOR_CODEX_REVIEW`를 기록한다.
4. Codex는 변경 diff, Actions 결과, 핸드오프 증거를 검수한 뒤에만 다음 작업 또는 병합을 승인한다.

GitHub Actions는 자동 테스트만 수행한다. 외부 AI의 완료를 감지해 Codex를 자동 기동하는 기능은 현재 연결되어 있지 않으므로, 검수 요청은 이 대화에서 명시해야 한다.
