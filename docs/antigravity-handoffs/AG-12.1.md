## 변경 파일
- `src/app/ledger/page.tsx`:
  - 단건 POST 대신 다수 행을 포함하는 배치 배열 형태로 `/api/ledger` 호출
  - '수기 세액' 체크박스 열을 추가하여 명시적으로 선택하지 않으면 서버의 중앙 금액 계산을 따르도록 하고(자동 세액) 세율 변경 시 세액이 자동 재계산되도록 UI/입력 모드 개선
  - 불필요한 `eslint-disable` 경고 제거를 위해 관련 주석 정리
- `src/app/api/ledger/route.ts`:
  - 다수 행을 전달받아 단일 데이터베이스 트랜잭션(`database.transaction`)을 통해 모두 저장하도록 배치 처리
  - 전달받은 배열 중 일부라도 계산/검증에 실패할 경우 0건이 저장되도록 처리
  - 서버 측에서 입력받은 `siteId`와 `costCategoryId`가 해당 `companyId`에 속하는지 엄격히 검증
  - 중복 제약 조건(unique constraint) 오류 발생 시 `409 CONFLICT` 안전 응답
  - 후행 공백 제거 완료
- `src/ledger/calculation.ts` 및 `src/ledger/calculation.test.ts`:
  - `isManualTax` 플래그가 true이더라도 `taxAmount`가 빈 값이면 fieldErrors에 포함시켜 저장을 차단하며 "0" 입력은 허용함
- `src/app/api/ledger/route.test.ts`:
  - `vi.clearAllMocks()` 사용 및 권한(`hasPermission`) 강제 모의 처리 적용
  - 수기 세액 누락에 따른 `[행번호].taxAmount` 필드 오류 및 `400` 반환 검증
  - 자동 세액 모드에서 `taxAmount`를 무시하고 `taxRate` 기준으로 계산하는 기능 검증 (트랜잭션 `mockTx`를 사용하여 실제 `insert`에 전달된 `taxAmount` 값을 정확히 캡처 후 `10.00` 검증)
  - 회사-비용분류 불일치 검증(`INVALID_CATEGORY_FOR_COMPANY`) 테스트 추가
  - 복수 유효 행 2건 저장 성공 테스트 추가 (트랜잭션 내 2건 `insert` 확인 및 반환 `records` 길이 검증)
  - 복수 행 처리 중 한 행이라도 오류 시 트랜잭션 진행 억제 검증

## 미실행 테스트
- E2E 테스트 혹은 브라우저 상의 직접 렌더링/인터랙션 테스트는 CLI 상에서만 수행하여 검증하지 않음.
- 세션 기반 실제 로그인 후 API 호출 시 인증/인가 통과 여부는 확인하지 않음.

## 미확인 업무 규칙
- 권한 검사, 세금 상세 규칙, 결재 및 지급 규칙에 대한 비즈니스 로직은 서버 단의 명세와 기존 `calculateMoney`에 전적으로 의존하며, 프론트엔드에서 임의로 추가하지 않음.
- 품명 중복, 수량/단가 한도 등에 대한 검사는 수행하지 않음.
